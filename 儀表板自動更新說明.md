# 讓儀表板「自己跑」＋付費一鍵抓取時一起更新

有兩種方式，可以只做一種，或兩種都做。

---

## 方式一：每天固定時間自動跑（推薦，不用開電腦）

用 **GitHub Actions** 排程，每天自動執行共識引擎，把最新的 `539_consensus_latest.json` 寫進 repo，儀表板就會自動更新。

### 步驟

1. **把這兩個放進 taiwan539-web 專案裡**  
   - `consensus_engine.py`（從 premium_backend 複製）  
   - 整個資料夾 `.github/workflows/`（裡面有 `refresh-consensus.yml`）

2. **在 GitHub 上 push**  
   ```bash
   cd /Users/user/taiwan539-web
   git add consensus_engine.py .github/workflows/refresh-consensus.yml
   git commit -m "加入儀表板自動更新排程"
   git push
   ```

3. **之後就會**  
   - 每天約 **台灣時間早上 10 點** 自動跑一次共識引擎  
   - 產生新的 `539_consensus_latest.json` 並自動 commit、push 到同一個 repo  
   - 儀表板讀的就是這個檔，所以**不用你手動跑、也不用開電腦**

4. **想立刻更新一次**  
   - 打開 GitHub → 你的 taiwan539-web repo → **Actions**  
   - 左邊點「更新儀表板共識數據」  
   - 右邊 **Run workflow** → Run workflow  
   - 跑完後儀表板就是最新數據

### 想改成「一天跑兩次」或改時間？

排程寫在 `.github/workflows/refresh-consensus.yml` 的 `schedule` 裡，用的是 **cron（UTC 時間）**。

- **現在**：`0 2 * * *` ＝ 每天 UTC 02:00 ＝ **台灣約早上 10:00 跑一次**。
- **改成台灣早上 10 點 + 晚上 6 點各跑一次**：多加一行，例如：
  ```yaml
  schedule:
    - cron: '0 2 * * *'   # 台灣 10:00
    - cron: '0 10 * * *'  # 台灣 18:00（UTC 10:00）
  ```
- **改時間**：台灣時間減 8 小時＝UTC。例如要台灣 **早上 8 點**：UTC 0 點 → `cron: '0 0 * * *'`。
- 改完存檔、push 到 GitHub 後，新排程就會生效。

## 方式二：付費用戶「一鍵抓取」時，順便更新共識數據

當有人用你平台的一鍵抓取時，**同時**觸發後端跑一次共識引擎，儀表板就會跟著更新。

**前提**：你要有一台**有在跑的後端**（例如用 FastAPI 的 main.py 部署在雲端或家裡電腦）。

### 後端（你已經有了）

- 在 **main.py** 裡已加上 **`POST /api/refresh-consensus`**  
- 呼叫一次就會執行共識引擎，並把結果寫成 `539_consensus_latest.json`  
- 若你的網頁和 API 都放在同一台主機，儀表板讀同一個目錄下的這個檔即可

### 前端（一鍵抓取時多打一個 API）

- 在 **index.html** 裡，付費使用者**一鍵抓取成功之後**，多送一個請求到你的後端：  
  `POST 你的後端網址/api/refresh-consensus`  
- 例如後端是 `https://你的主機.tw/api`，就多加一行（僅付費時）：  
  一鍵抓取成功後呼叫 `fetch('https://你的主機.tw/api/refresh-consensus', { method: 'POST' })`  
- 這樣每次有人用一鍵抓取，共識數據就會順便更新，儀表板就跟著新

### 若網站是純 GitHub Pages（沒有自己的後端）

- 就**沒有**「一鍵抓取時順便更新」的伺服器可以呼叫  
- 用 **方式一（GitHub Actions 排程）** 即可，儀表板一樣會每天自動更新

---

## 總結

| 做法 | 優點 | 需要什麼 |
|------|------|----------|
| **方式一：GitHub Actions 排程** | 全自動、不用開電腦、每天固定更新 | 把 `consensus_engine.py` 和 `.github/workflows/` 放進 taiwan539-web 並 push |
| **方式二：一鍵抓取時觸發** | 有人用就更新，更即時 | 要有部署好的 FastAPI 後端，並在首頁一鍵抓取成功後多呼叫 `POST /api/refresh-consensus` |

建議：**先做方式一**，儀表板就會自己每天更新；之後若有自己的後端，再加上方式二，就會變成「一鍵抓取時也一起更新」。

## 怎麼確認儀表板資料有沒有更新？

1. **看儀表板頁面**  
   打開「全網共識儀表板」，在標題「全網共識儀表板」**正下方**會有一行綠字：  
   **「數據更新時間：YYYY/MM/DD 上午/下午 HH:MM:SS」**  
   那就是這份資料的產生時間。若你剛跑完排程或手動更新，時間應該是最近幾分鐘內。

2. **看 GitHub Actions（有設排程的話）**  
   到 GitHub → 你的 taiwan539-web repo → **Actions**。  
   點「更新儀表板共識數據」，看**最後一次執行**的時間與結果（綠勾＝成功）。  
   有成功就會把新的 `539_consensus_latest.json` 推上去，儀表板讀的就是那份。

3. **直接看 JSON 檔**  
   在 GitHub 上打開 taiwan539-web 裡的 **539_consensus_latest.json**，找最上面的 **"generated_at"**，那一串就是資料產生時間（ISO 格式）。  
   若時間是最近的，就代表有更新。

## 時間一直沒變（例如卡在 12:38）怎麼辦？

- **先確認排程有沒有在跑**  
  儀表板每 3 小時更新一次的前提是：**taiwan539-web 這個 repo 裡要有** `consensus_engine.py` **和** `.github/workflows/refresh-consensus.yml`，並且有 push 上去。  
  → 到 GitHub → taiwan539-web → **Actions**，看「更新儀表板共識數據」**最近有沒有跑**（每 3 小時應至少跑一次）。若從沒跑過或失敗，請把 workflow 和 `consensus_engine.py` 都放進 taiwan539-web 再 push。

- **再確認是不是快取**  
  儀表板已改成每次開啟都會重新要一次 JSON（加時間參數避免快取）。  
  若 Actions 有在跑、但畫面上時間還是舊的，把手機或電腦的**強制重新整理**用一次（或關掉分頁再開一次儀表板）再看「數據更新時間」。
